GiftGuru

Project for Technology and Human Interaction (COMM 351)

Team Members: Prarthana Gupta, Matt Faden, Lucas Pettinato, Harish Raja, Minkyong Kim, Jimmy Yoon


Motivation

Gift-giving is a part of everyone's lives. Whether it is for birthdays, holidays, anniversies, or any other occasion, a gift is always part of the picture.

However, most people struggle when choosing and searching a gift for the receiver, even though they know the receiver well.

How can we facilitate this process for gift-givers?

Research Methodology

In order to better understand the context surrounding the gift-giving process, we conducted a contextual inquiry on a student who was interested in searching and buying a gift for Mother's Day.


The contextual inquiry revealed several pain points of gift-givers, such as:

  • Not knowing what to buy without directly asking the recipient
  • Not being able to search for product alternatives efficiently
  • Lack of information about ongoing discount and sales when online shopping


    • Having conducted our CI on a college student, we decided that it would appropriate to set our target user group as college student gift-givers (17-23 year olds).

    Prototype V1

    Some main features of the first version of the prototype included:

  • A home page
  • A sign-in page
  • A gift feed page displaying all the gift options
  • A wishlist
  • An option to specify details and search gifts for groups as recipients
  • An option to specify details and search gifts for male/female recipients

    User Testing

    Task Instructions:

    The following instruction set was given in three separate sets (Set A, B, C), repeated each time for a different recipient - male/female/group:

  • Task 1: Search for a gift for a male/female student/group

  • Task 2: Navigate to the Wish List.

  • Task 3: Navigate to your current orders

  • Task 4: Search for a gift for an existing user profile

    • Heuristic Evaluations

    Some of our group members also conducted heuristic evaluations to assess the usability of the interface. We wrote up our results as Usability Aspect Reports (UAR)s. An example can be found here.

    User performance was also evaluated for task completion, time, and errors. After testing, we measured satisfaction levels using 10-point scale items and open-ended questions (e.g. What would you change about the app?)

    Findings

  • Simple, uncluttered design
  • Relatively intuitive navigation
  • Broader options for various trait/interest/occasion categories beyond the built-in ones
  • button functionality (e.g. difficulty in clicking certain buttons)


    • ***Set A=male recipient task, Set B=female, Set C=group



    Iterations

      1. More feedback from user actions - Actions such as selecting description and interests had no feedback. When users tapped an icon, they were not sure whether it was registered by the system.
      2. Added user prevention - Instead of immediately removing items from cart upon tapping the 'x', a popup asking the user to confirm their action now appears.
      3. Increased button sizes - Items in the hamburger menu were increased.
      4. Onboarding process added - Meaning of symbols such as the heart or $ on the gift feed was unclear. Instruction bubbles were added the first time these symbols appeared to clear confusion.
