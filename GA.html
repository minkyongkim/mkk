Textbook Compare

Project for General Assembly User Experience Design Immersive


Motivation

As a college student myself, I have witnessed and experienced the challenge of buying textbooks for new courses every quarter. But what exactly makes this process so inconvenient and how can we improve it?

Problem Statement


College students have trouble finding and purchasing or renting textbooks for courses in a time and cost-efficient manner.


How can we simplify this process for students?

Research Methodology

<

In order to better understand the context and problems surrounding the textbook search process, I conducted a series of in-depth interviews with five college students. The main goals and subgoals of the interviews were:

    1. How do students currently search for textbooks online (or offline)?
  • What tools do they use? Websites, online stores, or others?
      • 2. When do students search for textbooks?
  • What time of the school year?
      • 3. How long do they spend on the search process on average?
      4. What are the various factors that influence the purchase decision?
  • Of the following possible factors, which do they consider? Are some more important than others?
  • Possible factors: shipping fee, delivery time, purchase options, condition of books, price range
      • 5. What are some common difficulties they experience in the process?
  • Which aspect is particularly difficult? What about the purchase process?


    • Findings:

    1. Majority searches online search engines (e.g. Google) for free online PDF versions

      2. If not available, uses online booksellers (e.g. Amazon, Chegg). Offline methods are last order of preference.

    2. Textbooks bought generally within first week of class, or until syllabus is out
    3. Roughly a few hours to a day, but depends on necessity of book
    4. Delivery time and price
    5. Search process requires visiting multiple booksellers (it never stops at just one)
    6. Inconvenience in filtering through results

    Paper Prototypes

    An initial prototype was sketched on paper and implemented on InVision.
    In designing the hi-fidelity prototype, I used the following techniques to help me guide my decisions.

    Feature Prioritization

    Another technique used before constructing the hi-fidelity prototype was feature prioritization. I prioritized features based on how important they were to addressing primary user needs revealed through responses from interviews and paper prototype testing. Based on the results, I decided that these features should be included in the prototype

  • Search tool for textbooks (by title, author, ISBN)
  • Dropdown menu for selecting universities
  • Price comparison tool
  • Shipping information display

    • Card Sorting

    A closed card-sort was created with five categories and 20 cards on OptimalSort, an online card-sorting tool, to describe the elements of app content. Two participants were given a link to the survey and asked to sort the 20 cards into given cateogries without a set time limit. More details here.

    User Testing

    Sample Demographics and Characteristics

    Ideal user sample varied on the following:

  • Age (17-23)
  • Attends a variety of colleges
  • Tech savvy nature
  • Usage and familiarity with mobile apps
  • Actual user sample: As a Northwestern University student, the most accessible sample was NU students.

    • Task-based User Testing

    Three sets of tasks were presented for each participant to complete:

      1. Task Set A: Search for a textbook and choose a bookseller
      2. Task Set B: Navigate to Purchase Checklist
      3. Task Set C: Navigate to My Account

    User testing sessions were recorded using the application Lookback (https://lookback.io) on Mac to capture think-aloud comments. The application captured both the users interacting with the prototype and phone screen as they navigated through the prototype on iPhone 6. See videos below.

    Results

    Problem #1: Counter-intuitive sequence

  • Users were confused by the "compare" button on the search results page
  • They did not know what was exactly being "compared"
  • Users were confused about having to search offline/online stores because search results generally display all existing books.


    • Problem #2: Placement of stars

  • Unclear whether stars indicated how expensive/cheap offered prices are


    • Problem #3: Two flows linking to purchase checklist

  • Users questioned differences between being able to manage checklists from My Account Settings page and accessing the checklist directly from the hamburger menu on the home page.


    • Iterations

  • To address Problem #1, a Book Description page providing full details about the book was inserted before Textbook Comparison page and after Search Results page for a more logical progression.

    • Compare buttons were removed from Search Results and relocated to Book Description page to indicate that booksellers are being compared for the same book.

  • To address #2, the word "rating" has been placed above stars to indicate that stars are representative of overall ratings of the bookseller based on trustworthiness, reliability, and other factors that contribute to quality of service. Price has been relocated to appear right below the bookseller's name.
  • To address #3, flow to the purchase checklist from Account Settings has been removed. Users can now only access the purchase checklist through the hamburger menu from the home page.
