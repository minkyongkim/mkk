OVERVIEW

General Assembly User Experience Design Immersive Project

PROJECT DESCRIPTION

A search and comparison tool to help college students find good textbook deals

ROLE

Flow analysis, prototyping, usability testing, heuristic evaluation

>h4>TOOLS

Justinmind, OptimalSort, Lookback.io

DURATION

June - August 2016

MOTIVATION

Many college students have trouble finding and purchasing or renting textbooks for courses in a time and cost-efficient manner.
How can we simplify this process for students?

RESEARCH METHODOLOGY

In order to better understand the context and problems surrounding the textbook search process, I conducted a series of in-depth interviews with five college students. The main goals and subgoals of the interviews were:

  1. How do students currently search for textbooks online (or offline)?
  2. What tools do they use? Websites, online stores, or others?
  3. What time of the school year do students search for textbooks?
  4. How long do they spend on the search process on average?
  5. What are the various factors that influence the purchase decision?
  6. Of the following possible factors e.g. shipping fee, delivery time, etc., which do they consider? Are some more important than others?
  7. What are some common difficulties they experience in the process? Which aspect is particularly difficult?

Findings:

  1. Majority searches online search engines (e.g. Google) for free online PDF versions. If not available, uses online booksellers (e.g. Amazon, Chegg). Offline methods are last order of preference.
  2. Textbooks bought generally within first week of class, or until syllabus is out
  3. Roughly a few hours to a day, but depends on necessity of book
  4. Delivery time and price
  5. Search process requires visiting multiple booksellers (it never stops at just one)
  6. Inconvenience in filtering through results

Personas

Interview insights were then translated into personas to paint a clearer picture of target users.


COMPETITIVE ANALYSIS

To learn more about what features seem to be essential for textbook comparison technologies, I conducted analysis on the following products:

  • BookFinder
  • Textsurf
  • Bigwords.com

Each was analyzed for UI/ease of use, layout and design, qualities, and areas of improvement

Summary:

  • All were easy to use

But could improve on:

  • Layout- cluttered, better use of space
  • Displaying essential information concisely
  • Better named categories (lack of information scent)

PROTOTYPES

User Flow


Paper Prototypes

An initial prototype was sketched on paper and implemented on InVision.
To design the hi-fidelity prototype, I used the following techniques to help me guide my decisions.

Feature Prioritization

Another technique used before constructing the hi-fidelity prototype was feature prioritization. I prioritized features based on how important they were to addressing primary user needs revealed through responses from interviews and paper prototype testing. This helped to build a prototype with minimal, but the most essential features to test on users. I decided that the critical features were:

  • Search tool for textbooks (by title, author, ISBN)
  • Dropdown menu for selecting universities
  • Price comparison tool
  • Shipping information display

Card Sorting

A closed card-sort was created with five categories and 20 cards on OptimalSort, an online card-sorting tool, to help guide designing of the information architecture. Each card represented either a high-level modules or subcategories. Two participants were given a link to the survey and asked to sort the 20 cards into given cateogries without a set time limit. USER TESTING

Sample Demographics and Characteristics

Ideal user sample varied on the following:

  • Age (17-23)
  • Attends a variety of colleges
  • Tech savvy nature
  • Usage and familiarity with mobile apps

Actual user sample: As a Northwestern University student, the most accessible sample was NU students.


Task-based User Testing

Three sets of tasks were presented for each participant to complete:

  1. Task Set A: Search for a textbook and choose a bookseller
  2. Task Set B: Navigate to Purchase Checklist
  3. Task Set C: Navigate to My Account

RESULTS

Problem #1: Counter-intuitive sequence

  • Users were confused by the "compare" button on the search results page
  • They did not know what was exactly being "compared" (no price information was visible).
  • Users were confused about having to search offline/online stores because search results generally display all existing books.

Problem #2: Placement of stars

  • Unclear whether stars indicated how expensive/cheap offered prices are

Problem #3: Two flows linking to purchase checklist

  • Users questioned differences between being able to manage checklists from My Account Settings page and accessing the checklist directly from the hamburger menu on the home page.

The 2nd Iteration

To address Problem #1, a Book Description page providing full details about the book was inserted before Textbook Comparison page and after Search Results page for a more logical progression. Compare buttons were removed from Search Results and relocated to Book Description page to indicate that booksellers are being compared for the same book.

To address #2, the word "rating" has been placed above stars to indicate that stars are representative of overall ratings of the bookseller based on trustworthiness, reliability, and other factors that contribute to quality of service. Price has been relocated to appear right below the bookseller's name.

To address #3, flow to the purchase checklist from Account Settings has been removed. Users can now only access the purchase checklist through the hamburger menu from the home page.

TAKEAWAYS

Don't let symbols speak for themselves

Users might be able to guess what stars and a "compare" button next to a textbook mean, but can be confused if they are not shown in a contextually relevant manner. Symbols and CTA's are only meaningful if accompanied by other elements that help explain them


A mixed methods approach can be powerful

Using a mixed methods approach to solve problems can help make better design decisions if done correctly.
